# Blockchain & Web3 Technical Capabilities Reference

> Source: `sample/blockchain-web3/agents/blockchain-developer.md`

## Smart Contract Development

### Solidity & EVM
- **Advanced patterns**: Proxy contracts, diamond standard, factory patterns
- **Upgradeable contracts**: Transparent, UUPS, beacon proxies
- **Security**: Reentrancy, overflow, access control vulnerability prevention
- **Libraries**: OpenZeppelin integration for battle-tested contracts
- **Optimization**: Gas optimization, contract size minimization
- **Verification**: Formal verification (Certora, Slither, Mythril)

### Multi-Chain Development
- **Rust contracts**: Solana (Anchor), NEAR, Cosmos SDK
- **Vyper**: Enhanced security, formal verification
- **Cross-chain**: Bridge development, security considerations

## Ethereum Ecosystem

### Layer 1 & Layer 2
- **Web3 libraries**: Web3.js, Ethers.js, Viem
- **L2 solutions**: Polygon, Arbitrum, Optimism, Base, zkSync
- **EVM chains**: BSC, Avalanche, Fantom integration

### Token Standards
- **ERC-20**: Fungible tokens
- **ERC-721**: NFTs
- **ERC-1155**: Multi-token standard
- **ERC-4337**: Account abstraction

### Advanced Concepts
- **MEV protection**: Front-running prevention, flashloan strategies
- **Staking**: ETH 2.0 validator operations
- **Cross-chain bridges**: Security considerations

## DeFi Protocol Development

- **AMMs**: Uniswap V2/V3, Curve, Balancer mechanics
- **Lending**: Compound, Aave, MakerDAO patterns
- **Yield farming**: Liquidity mining, auto-compounding
- **Derivatives**: Perpetual swaps, options protocols
- **Flash loans**: Implementation, arbitrage strategies
- **Governance**: DAOs, token voting, treasury management
- **Insurance**: Decentralized risk assessment protocols
- **Synthetics**: Oracle integration, asset tokenization

## NFT & Digital Assets

- **Standards**: ERC-721, ERC-1155, royalties (EIP-2981)
- **Marketplaces**: OpenSea-compatible contracts
- **Generative art**: On-chain metadata, IPFS storage
- **Utility**: Gaming, membership, governance integration
- **Fractional ownership**: Tokenization patterns
- **Dynamic NFTs**: Oracle integration, time-based mechanics

## Web3 Frontend

- **Wallet integration**: MetaMask, WalletConnect, Coinbase Wallet
- **React frameworks**: Wagmi, RainbowKit, Web3Modal
- **UX patterns**: Gasless transactions, meta-transactions, relayers
- **Authentication**: Web3 session management, DID
- **Mobile**: React Native Web3, mobile SDKs

## Infrastructure & DevOps

### Development Tools
- **Local blockchain**: Hardhat, Foundry, Ganache
- **Testing**: Unit tests, fuzzing, property-based tests
- **Deployment**: Testnet, mainnet, multi-chain automation

### Indexing & Data
- **The Graph Protocol**: Subgraph development
- **Custom indexers**: Event processing pipelines
- **RPC management**: Node load balancing
- **IPFS**: Decentralized storage, pinning services

## Oracles & External Data

- **Chainlink**: Price feeds, VRF, Functions
- **Alternative oracles**: API3, Band Protocol, Pyth Network
- **Custom oracles**: Data source integration
- **Security**: MEV protection, front-running prevention

## Tokenomics & Economics

- **Distribution**: Vesting schedules, airdrops
- **Pricing**: Bonding curves, dynamic mechanisms
- **Staking rewards**: Calculation, distribution
- **Governance**: Token voting, proposal execution
- **Treasury**: Protocol-owned liquidity
- **Deflationary models**: Token burning mechanisms

## Enterprise Blockchain

- **Private networks**: Consortium chains, Hyperledger Fabric
- **Supply chain**: Tracking, verification systems
- **Identity**: KYC/AML compliance, digital identity
- **CBDC**: Central Bank Digital Currency integration
- **Tokenization**: Real estate, commodities, securities
- **Compliance**: Regulatory frameworks, reporting

## Security & Auditing

- **Vulnerability assessment**: Penetration testing
- **Key management**: Hardware wallets, multi-sig
- **Cryptography**: zk-SNARKs, zk-STARKs, threshold signatures
- **Forensics**: Transaction analysis, incident response
- **Monitoring**: Anomaly detection, security alerts
