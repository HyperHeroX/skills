## Usage

These capability files are referenced in the Job Description (JD) files to provide detailed technical context for each role. When simulating a role or reviewing code, consult the relevant capability files for:

1. **Technical depth**: Understand tools, frameworks, and best practices
2. **Quality standards**: Apply the documented patterns and anti-patterns
3. **Knowledge areas**: Cover all relevant domains when planning or reviewing

## File Structure

Each capability file follows this structure:

```markdown
# [Domain] Technical Capabilities Reference

> Sources: [Sample agent files used]

## [Topic 1]
- Key concepts
- Tools and frameworks
- Best practices

## [Topic 2]
- Tables for comparisons
- Code examples where relevant
- Anti-patterns to avoid
```
