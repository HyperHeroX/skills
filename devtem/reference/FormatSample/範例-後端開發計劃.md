## 資料庫設計

### 主要資料表
1. **Users**: 用戶資訊
2. **Posts**: 貼文資訊
3. **Comments**: 回覆資訊
4. **Images**: 圖片資訊
5. **Tags**: 標籤資訊
6. **PostTags**: 貼文-標籤多對多關聯
7. **PostImages**: 貼文-圖片多對多關聯
8. **CommentImages**: 回覆-圖片多對多關聯

## 開發順序與依賴關係

1. **第一階段**: Auth 模組 (用戶認證是所有功能的基礎)
2. **第二階段**: Image 模組 (圖片上傳是貼文和回覆的必要功能)
3. **第三階段**: Post 模組 (貼文是回覆的容器)
4. **第四階段**: Comment 模組 (回覆依賴於貼文)

## 安全考量

- 實現 JWT 認證與授權機制
- 防止 XSS 攻擊 (清理用戶輸入)
- 防止 CSRF 攻擊 (使用 Anti-Forgery Token)
- 圖片上傳安全 (檢查文件類型、大小、移除 EXIF 資訊)
- 資料庫查詢優化 (防止 SQL 注入、避免 N+1 查詢問題)
- 權限檢查 (確保只有作者和管理員能編輯/刪除內容)

## 效能考量

- 資料庫索引優化
- 使用延遲載入避免 N+1 查詢問題
- 圖片壓縮與 CDN 分發
- API 回應快取
- 分頁查詢優化
