# 資料庫設計

本文檔詳細描述專業模擬考試系統的資料庫架構設計，包括關係模型、實體關係圖、索引策略、查詢優化以及資料流分析。

## 資料庫架構概述

專業模擬考試系統採用混合資料庫架構，使用關係型資料庫作為主要數據存儲，結合文檔型資料庫處理靈活內容，以及向量資料庫支持AI檢索，同時使用時序資料庫記錄學習行為與系統指標。

### 資料庫技術選擇

1. **主要關係型資料庫**：PostgreSQL 15+
   - 用於結構化數據與核心業務邏輯
   - 支持JSON數據類型與全文檢索
   - 強大的事務處理與數據一致性

2. **文檔資料庫**：MongoDB
   - 用於存儲考題內容與靈活學習資源
   - 支持複雜嵌套文檔與動態架構
   - 高寫入性能與水平擴展能力

3. **向量資料庫**：Pinecone
   - 存儲嵌入向量用於AI檢索
   - 支持語義搜索與相似度匹配
   - 高效向量索引與查詢優化

4. **時序資料庫**：TimescaleDB (PostgreSQL擴展)
   - 記錄用戶學習行為與系統性能指標
   - 高效時間序列數據查詢
   - 資料自動分區與保留策略

5. **緩存層**：Redis
   - 用戶會話管理
   - 熱點數據緩存
   - 分佈式鎖與速率限制
   - 實時統計與排行榜

## 核心數據模型

### 用戶模型

#### Users 表（用戶基本信息）
```sql
CREATE TABLE Users (
    user_id UUID PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    salt VARCHAR(50) NOT NULL,
    full_name VARCHAR(100),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    active BOOLEAN DEFAULT TRUE,
    verified BOOLEAN DEFAULT FALSE,
    role_id INT REFERENCES Roles(role_id),
    avatar_url VARCHAR(255),
    last_login_at TIMESTAMP WITH TIME ZONE,
    login_count INT DEFAULT 0,
    timezone VARCHAR(50) DEFAULT 'UTC'
);
```

CREATE TABLE UserProfiles (

### 考試模型

#### Exams 表（考試定義）
```sql
CREATE TABLE Exams (
    exam_id UUID PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    category_id UUID REFERENCES ExamCategories(category_id),
    creator_id UUID REFERENCES Users(user_id),
    difficulty_level SMALLINT CHECK (difficulty_level BETWEEN 1 AND 5),
    time_limit_minutes INT,
    passing_score DECIMAL(5,2),
    is_public BOOLEAN DEFAULT FALSE,
    question_count INT DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'draft',
    version INT DEFAULT 1,
    ai_generated BOOLEAN DEFAULT FALSE,
    rag_knowledge_base_id UUID REFERENCES KnowledgeBases(kb_id)
);
```

#### Questions 表（考題）
```sql
CREATE TABLE Questions (
    question_id UUID PRIMARY KEY,
    question_type VARCHAR(50) NOT NULL,
    content JSONB NOT NULL,
    correct_answer JSONB NOT NULL,
    explanation TEXT,
    difficulty_level SMALLINT CHECK (difficulty_level BETWEEN 1 AND 5),
    creator_id UUID REFERENCES Users(user_id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'active',
    ai_generated BOOLEAN DEFAULT FALSE,
    rating DECIMAL(3,2),
    rating_count INT DEFAULT 0,
    usage_count INT DEFAULT 0
);
```

#### ExamQuestions 表（考試與考題關聯）
```sql
CREATE TABLE ExamQuestions (
    exam_question_id UUID PRIMARY KEY,
    exam_id UUID REFERENCES Exams(exam_id) ON DELETE CASCADE,
    question_id UUID REFERENCES Questions(question_id) ON DELETE CASCADE,
    question_order INT NOT NULL,
    points DECIMAL(5,2) DEFAULT 1.0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

#### UserExams 表（用戶考試記錄）
```sql
CREATE TABLE UserExams (
    user_exam_id UUID PRIMARY KEY,

    start_time TIMESTAMP WITH TIME ZONE,
    finish_time TIMESTAMP WITH TIME ZONE,

## 數據流設計

### 用戶學習數據流

1. **考題回答 → 知識點更新流程**
   - 用戶回答考題後，通過QuestionKnowledgePoints關聯表識別相關知識點
   - 更新UserKnowledgePointProgress表中的正確/錯誤計數
   - 使用衰減算法重新計算知識點熟練度
   - 基於熟練度調整下次複習時間

2. **學習計劃 → 資源推薦流程**
   - 分析用戶知識點薄弱區域
   - 基於UserKnowledgePointProgress數據選擇重點知識點
   - 通過KnowledgePointResources關聯查詢相關學習資源
   - 為StudyPlanItems表生成個性化學習項目

3. **用戶行為 → 學習分析流程**
   - 記錄用戶學習行為到UserLearningEvents時序表
   - 按時間窗口聚合分析學習模式
   - 更新UserInsights表中的學習效率指標
   - 生成個性化學習建議

### AI生成數據流

1. **考題生成流程**
   - 從KnowledgeBases中檢索相關知識點資料
   - 使用AIModels表中配置的生成模型創建考題
   - 存儲生成結果到Questions表，設置ai_generated=true
   - 通過AI知識點標記模型關聯相關知識點

2. **學習資源生成流程**
   - 基於UserKnowledgePointProgress識別需要加強的知識點
   - 結合KnowledgePointRelations表的關聯知識分析
   - 使用適合的AI模型生成學習資源
   - 存儲到LearningResources表並建立知識點關聯

## 索引與優化策略

```

### 查詢優化策略

1. **批量用戶學習數據查詢優化**
   - 使用物化視圖預先聚合用戶學習統計
   - 實現資料分區策略(按時間/用戶ID)
   - 設置適當查詢緩存策略

2. **知識點關聯數據查詢優化**
   - 使用遞歸CTE高效查詢知識點樹結構
   - 針對常用知識點路徑建立專用索引
   - 考慮知識圖譜存儲與查詢優化

3. **考試數據查詢優化**
   - 設計高效的試題選擇算法
   - 考試結果分析採用增量更新策略
   - 使用批量插入減少考試記錄寫入壓力

## 資料倉庫設計

為支持高級分析和報告需求，系統設計包含獨立的資料倉庫層：

### 事實表

1. **UserExamFact** - 考試事實表
   - 維度：用戶、考試、日期、時間、設備
   - 度量：分數、時間花費、題目數量、正確率

2. **UserLearningFact** - 學習行為事實表
   - 維度：用戶、資源類型、知識點、日期
   - 度量：學習時間、完成度、互動次數

3. **KnowledgePointProgressFact** - 知識點掌握度事實表
   - 維度：用戶、知識點、課程、時間
   - 度量：熟練度變化、測試次數、複習頻率

### 維度表

1. **UserDim** - 用戶維度
2. **TimeDim** - 時間維度
3. **ExamDim** - 考試維度
4. **KnowledgePointDim** - 知識點維度
5. **ResourceDim** - 資源維度

### ETL流程

1. **數據提取**：每日增量同步操作數據
2. **數據轉換**：維度表更新、度量標準化
3. **數據加載**：更新數據倉庫與OLAP立方體
4. **報表更新**：刷新標準報表與儀表板

## 資料遷移與維護計劃

### 版本控制策略

- 採用數據庫版本控制工具（如Flyway、Liquibase）
- 每個遷移腳本單一職責原則
- 向前/向後兼容性考量

### 數據遷移流程

1. **Schema變更測試**：在測試環境驗證
2. **數據一致性檢查**：數據遷移前後完整性檢驗
3. **增量部署策略**：小批量、低風險部署
4. **回滾計劃**：緊急回退措施設計

### 資料庫維護計劃

1. **性能監控**：定期檢查關鍵指標與慢查詢
2. **索引維護**：周期性索引重建與優化
3. **數據清理**：過期數據存檔與清理政策
4. **容量規劃**：預測增長需求的擴展計劃
