
# 職位名稱：系統分析師（System Analyst）

## 📌 角色定位
系統分析師為連接業務需求與技術實作的核心角色，負責需求分析、規格撰寫、系統設計與開發協作。其核心任務在於將使用者需求轉化為具體、可開發的技術規格，並確保在系統開發生命週期中維持品質、效率與邏輯一致性。

---

## 🛠️ 核心技能要求

### 1. 業務需求分析
熟悉需求蒐集技術（訪談、工作坊、問卷等），可撰寫清晰的 SRS 或使用者故事。

### 2. 系統設計與資料建模
能運用 UML、ERD、DFD 等工具，進行系統功能拆解與資料流程設計。

### 3. 溝通與跨部門協作
能作為 PO、技術團隊、測試人員之間的橋樑，有效轉譯業務與技術語言。

### 4. 規格文件撰寫
熟悉撰寫技術規格書、API 文件、使用流程與介面定義。

### 5. 測試驗收支援
能協助撰寫驗收標準與測試案例，支援 UAT 測試與問題追蹤。

---

## 🧩 在專案中的工作描述
- 在專案啟動階段與業主確認業務需求與流程  
- 撰寫 BRD、SRS、Use Case 與流程圖  
- 協助 UI/UX 團隊設計原型與交互流程  
- 與後端團隊定義 API 規格與資料結構  
- 跟進進度、參與開發會議並解釋需求背景  
- 撰寫測試案例並支援 QA 測試與驗收  

---

## ⭐ 加強技能（進階加分條件）

### 跨產業知識
具備金融、教育、電商、製造等垂直領域經驗，可快速理解業務語境。

### 敏捷方法導入
熟悉 Scrum 流程，能主導 Backlog Refinement 與 Sprint Planning。

### DevOps 觀念
了解 CI/CD、版本控管（Git Flow）與自動化測試流程。

### 工具熟悉度
熟練使用 Jira、Confluence、Draw.io、Swagger、Postman 等分析與協作工具。

### 技術素養
具備基本程式邏輯能力，能閱讀前端框架與後端語言（如 JavaScript、Python、C#）。

### 英文文件理解能力
能閱讀國際標準或第三方 API 文件，快速整合技術資料。

---

## 📐 編輯規則建議（供 AI 系統使用）
- 保持條列清晰、語句結構簡潔  
- 每項技能說明以一句完整句子定義核心行為或價值  
- 所有專案描述需以「能夠」「負責」「協助」等主動語態撰寫  
- 避免使用含糊詞語（如「了解」、「接觸過」），改以具體動作表達
