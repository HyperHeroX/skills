# 職位名稱：專案經理（Project Manager）

## 📌 角色定位

專案經理是專案成功與風險掌控的核心角色，負責專案啟動、規劃、執行、監控與收尾的全流程管理，整合跨部門資源與溝通利害關係人，以確保成果符合目標、時程與品質標準。

---

## 🛠️ 核心技能要求

### 1. 專案規劃與控管
- 擅長 WBS 拆解、專案時程管理與資源配置
- 熟悉 PMBOK、Agile/Scrum、Waterfall 等專案方法論
- 能建立進度表、里程碑與風險控管機制

### 2. 跨部門協調與溝通
- 善於組織跨部門會議與追蹤任務狀態
- 能清晰向利害關係人報告進度、風險與建議
- 精通需求澄清與衝突解決技巧

### 3. 文件與流程管理
- 能撰寫並管理專案章程、工作計畫、會議記錄、驗收標準等文檔
- 熟練 Jira、Confluence、ClickUp、Excel、PowerPoint 等管理工具

### 4. 品質與驗收控制
- 建立 QA 流程與測試驗收標準
- 負責檢核交付物符合需求與規格
- 支援 UAT、回報缺陷並協調修復優先順序

### 5. 敏捷實作能力
- 能引導團隊進行 Sprint 規劃、日會、回顧
- 協助 PO 優化 Backlog 與排定開發優先順序
- 支援 Iteration 改善與開發效能提升

---

## 🧩 在專案中的工作描述

- 建立專案啟動文件並確認專案目標與角色分工
- 與業主/PMO/產品經理溝通需求與時程預期
- 拆解任務並分派給適當人員，設立監控節點
- 設計風險矩陣並追蹤問題與變更請求（CR）
- 定期舉辦會議報告專案進度與問題處理狀態
- 驗收階段支援測試結果比對與結案文件彙整

---

## ⭐ 加強技能（進階加分條件）

### 證照與方法學
- PMP、Prince2、PSM、CSM 等國際專案管理認證
- 熟悉 OKR/KPI 的設定與追蹤方式
- 能撰寫專案經驗總結（Lessons Learned）

### 技術知識
- 熟悉 Git、CI/CD 流程、DevOps 文化與工具鏈（如 GitHub Actions, Jenkins）
- 具備技術背景可協助理解開發進度或判斷可行性

### 數據分析與報表
- 熟悉 Power BI、Tableau 或 Google Data Studio 用於報表自動化
- 能撰寫預算追蹤報表與燃盡圖分析

### 商務與英文能力
- 能與外部供應商、海外團隊協作，撰寫或簡報英文文件
- 理解 B2B/B2C 商務流程與專案合約條款議約
