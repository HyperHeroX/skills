# /repage — Page-Specific Presentation Editor

Modify a specific page (or page range) in an existing HTML presentation generated by the `interactive-presentation` skill. The command guides the user through structured Q&A and composes an optimized modification prompt following the 8 prompt design principles.

## Usage

```
/repage                              # Full Q&A flow
/repage <file-path>                  # Skip file question
/repage <file-path> <page-number>    # Skip file and page questions
```

## Required Context

Before running, the `interactive-presentation` skill should be loaded. This command relies on:
- `references/prompt-engineering-guide.md` — the 8 design principles and scenario templates
- `references/prompt-library.md` — 60+ categorized prompts
- `references/workflow-guide.md` — animation keywords and template guides

---

## Execution Flow

### Phase 1: Target Identification

Ask in this order, one question at a time (do not batch):

**Q1. Target File**
```
Which HTML presentation file do you want to modify?
(Provide the full path, or press Enter to use the default: Desktop/AI_Animation.html)
```
- If skipped or empty: default to `Desktop/AI_Animation.html`
- Verify the file exists via Read. If not found, ask again.

**Q2. Page Scope**
```
Which page(s) do you want to modify?
  • Single page:   "3"      (modify page 3 only)
  • Range:         "3-5"    (modify pages 3 to 5)
  • Multiple:      "1,3,7"  (modify specific pages)
  • All pages:     "all"    (global modification)
```

**Q3. Preview the Target**
- Read the target file
- Extract page structure (usually `<section class="slide">` or similar)
- Summarize what's currently on each target page in 1-2 sentences
- Confirm with user: "Is this the correct page? (yes/no)"
- If no: re-ask page number

---

### Phase 2: Change Type Classification

Present the change type menu:

```
What type of modification do you want?

  [1] Content          — Edit text, data, facts, headings
  [2] Layout           — Rearrange elements, change grid/flex structure
  [3] Animation        — Adjust entrance, timing, sequencing, emphasis
  [4] Visual Style     — Colors, theme (dark/light), accent changes
  [5] Typography       — Font size, weight, spacing, emphasis marks
  [6] Icons            — Replace emoji with flat UI icons, swap icon set
  [7] Interactive FX   — Hover, glow, 3D rotation, particles
  [8] Template Style   — Apply a different PPT/RNN template
  [9] Add / Remove     — Add new element or remove existing one
  [10] Multiple        — Combination (specify which)
```

Based on selection, proceed to the relevant Phase 3 branch.

---

### Phase 3: Requirement Collection (Branch by Change Type)

Apply the 8 design principles contextually — only ask what's relevant to the change type.

#### Branch [1] Content

- What is the new content? (paste or describe)
- Does the new content fit the "large text, few words, many graphics" principle?
  - If content is >100 words per page: recommend splitting into 2 pages or converting to visual elements
- Should existing animations continue to work? (default: yes, preserve sequencing)

#### Branch [2] Layout

- What layout structure do you want? Options:
  - Side-by-side comparison (2-column)
  - Top-bottom stack
  - Centered hero + details below
  - Grid (2x2, 3x3)
  - Timeline (vertical/horizontal)
  - Card-based layered (reference: `assets/examples/Animation/RNN-3.html`)
- Should the layout match one of the bundled examples? (offer to read `assets/examples/`)
- Will the layout work for video recording (large, readable, uncluttered)?

#### Branch [3] Animation

Apply the **Animation Sequencing Discipline** (Principle #4):
- What's the entrance order? (top→bottom / left→right / center outward / custom sequence)
- Total animation duration for this page? (default: 5 seconds)
- Should parent modules appear before child elements? (default: yes, hierarchical)
- Any persistent effects after entrance? (pulse, glow, flow arrows)
- Should transitions between previous/next pages have special effects? (fade, slide, 3D)

Reference: `references/prompt-library.md` Section 2 (Animation Effects, 16 prompts)

#### Branch [4] Visual Style

Apply the **Dark Theme + Accent** principle (Principle #8):
- Background: Dark (#0a0a0f) / Light (white) / Custom hex?
- Primary accent color? (cyan, purple, pink, yellow, green, custom hex)
- Secondary accent? (optional)
- Background effects? (particle canvas, noise texture, gradient radial)
- Reference an example file for style matching? (offer to read from `assets/examples/BG/`)

#### Branch [5] Typography

Apply the **Large Text, Few Words** principle (Principle #3):
- Body text size? (default: 24px minimum)
- Heading size? (default: 48px minimum)
- Emphasis marks to apply? Pick any:
  - Bold
  - Underline
  - Italic
  - Text color (which?)
  - Text background highlight (which color?)
- Line spacing adjustment? (default: 1.6)

#### Branch [6] Icons

Apply the **Emoji Prohibition** principle (Principle #6):
- Icon library preference? (Font Awesome, Material Icons, Lucide, inline SVG)
- Icon style? (solid, outline, duotone)
- Any specific icons to use for specific concepts?

If any emoji are detected on the target page, automatically flag them for replacement.

#### Branch [7] Interactive FX

- Hover effects? (glow, scale, color shift, lift)
- 3D effects? (mouse-follow rotation, auto-rotation, perspective transform)
  - If 3D: specify angle range and cycle duration
- Particle background? (density, color, speed)
- Glow/bloom intensity? (subtle / medium / strong)

Reference: `references/prompt-library.md` Section 3 (3D Effects, 4 prompts) and Section 6 (Beautification)

#### Branch [8] Template Style

Apply the **Template-Driven 95% Fidelity** principle (Principle #5):
- Which template to apply?
  - PPT-Generate-1.html — Clean modular
  - PPT-Generate-2.html — Chart-rich
  - **PPT-Generate-3.html — Best visual (recommended)**
  - PPT-Generate-4.html — Flexible layout
  - RNN-2~7.html — Flowchart styles
- Mandatory: read the template file before composing the prompt
- Mandatory: include "replace all emoji with flat UI library icons"
- Mandatory: include "target 95%+ visual similarity with template"

#### Branch [9] Add / Remove

For add:
- What element to add? (text, image, icon, chart, diagram, custom)
- Where on the page? (top/middle/bottom/left/right/specific position)
- Entrance animation for the new element? (default: ease-in matching page style)

For remove:
- Which element? (confirm by quoting its current text/role)
- Should animation sequence renumber automatically? (default: yes)

#### Branch [10] Multiple

Repeat relevant branches in sequence. Collect all requirements before composing the prompt.

---

### Phase 4: Prompt Composition

Compose a compound modification prompt following the rules from `references/prompt-engineering-guide.md`:

**Rule 1: Compound Prompts** — combine all requirements into one prompt
**Rule 2: Negative Constraints** — preserve unchanged pages ("only modify page X, do not alter other pages")
**Rule 3: Reference Before Restructuring** — read template file if applicable
**Rule 4: Quantify** — use specific numbers (24px, 5s, 95%, 1000 lines)
**Rule 5: Iterative Refinement** — target only the specified scope

**Template structure:**

```
Modify ONLY page [N] (or pages [range]) of the file: [full-path]

Change type: [type from Phase 2]

Requirements:
- [requirement 1 from Phase 3, quantified]
- [requirement 2 from Phase 3, quantified]
- ...

Constraints (mandatory):
- Do NOT alter other pages or the overall presentation structure
- Preserve existing keyboard navigation and page-transition logic
- Preserve animation hierarchy (parent → child)
- Replace any emoji with flat UI library icons
- Maintain single-file, zero-dependency architecture
- Keep the same color palette unless a color change was requested
- Ensure the modified page still works at 100vw × 100vh viewport

Visual quality bar:
- Large text: minimum 24px body, 48px headings
- Animation timing: ease-in sequential, total 5 seconds per page
- Video-friendly: readable at 1080p playback, reduced text, rich graphics
- [If template applied] Target 95%+ visual similarity with [template-path]

Output: update the file in place at [full-path]
```

Show the composed prompt to the user for confirmation before execution.

---

### Phase 5: Preview & Confirm

Before modifying the file:

```
📋 Proposed Modification Summary

File:    [path]
Page(s): [N or range]
Type:    [change type]
Changes:
  • [change 1]
  • [change 2]
  • ...

🎯 Composed Prompt Preview:
[show composed prompt]

Proceed? (yes / no / edit)
```

- **yes** → Execute modification (Phase 6)
- **no** → Abort
- **edit** → Return to Phase 3 to adjust

---

### Phase 6: Execution

1. Read the target file
2. Apply the composed prompt to regenerate only the target page(s)
3. Preserve all other pages verbatim
4. Use Edit tool with precise old_string/new_string for targeted page replacement
5. If the change spans too much of the file: use Write with full updated content
6. Report completion:

```
✅ Modification applied

File:    [path]
Pages:   [N]
Changes: [summary]

Next steps:
- Open the file in a browser to verify
- Use /repage again for additional modifications
- Use the `interactive-presentation` skill for new presentations
```

---

### Phase 7: Post-Modification Validation (Auto)

Automatically verify the 8 principles were preserved:

| Principle | Check |
|-----------|-------|
| Video-First | Text size ≥ 24px body, reduced text per slide |
| Progressive Enhancement | Only the specified scope was modified |
| Large Text Few Words | No paragraphs over 30 words per visual block |
| Animation Sequencing | All elements on modified page have animation-delay |
| Template Fidelity | If template applied, key style markers match |
| Emoji Prohibition | No `emoji` characters in modified page |
| Single-File | No new external dependencies added |
| Dark Theme + Accent | Color scheme preserved unless explicitly changed |

Report any violations found and offer to auto-fix.

## Examples

### Example 1: Change animation timing on page 3

```
User: /repage ~/Desktop/AI_Animation.html 3

AI: [reads file, previews page 3]
    "Page 3 shows the TCP three-way handshake diagram. Correct? (yes/no)"

User: yes

AI: "What type of modification?"
    [shows menu]

User: 3 (Animation)

AI: "What's the current issue with the animation?"

User: "Elements appear too fast — hard to follow in a video"

AI: "Current total duration seems to be 3s. Recommended: 5s for video pacing.
     Should I apply: 5-second total entrance, hierarchical parent→child sequence,
     ease-in curve?"

User: yes

AI: [composes prompt, shows preview, executes]
```

### Example 2: Apply template to pages 5-7

```
User: /repage

AI: "Which file?"
User: ./my-presentation.html

AI: "Which pages?"
User: 5-7

AI: [previews pages 5-7]
AI: "What type of modification?"
User: 8 (Template Style)

AI: "Which template? (shows PPT and RNN options)
     Recommended for these pages: RNN-3.html (layered cards) if they show processes,
     or PPT-Generate-3.html (best visual) for general content."

User: RNN-3

AI: [reads RNN-3.html template, composes 95%-fidelity prompt, applies to pages 5-7]
```

## Design Philosophy Enforcement

This command enforces the 8 principles at every step:

1. **Video-First** — Q&A asks about video context, prompts include "for video demonstration"
2. **Progressive Enhancement** — Only modifies specified pages, preserves rest
3. **Large Text Few Words** — Validates text size and word count in Phase 7
4. **Animation Sequencing** — Branch [3] enforces hierarchical ordering
5. **Template Fidelity** — Branch [8] mandates reading template + 95% target
6. **Emoji Prohibition** — Branch [6] + Phase 7 validation
7. **Single-File** — Phase 7 checks no new external deps added
8. **Dark Theme + Accent** — Branch [4] preserves palette unless changed
