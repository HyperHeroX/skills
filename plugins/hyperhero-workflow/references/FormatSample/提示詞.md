# 提示詞

## 1. 概述產品內容
### 進行產品功能討論
我現在想為系統的使用者建立一個 DashBoard ，但我不知道什麼樣的資訊是這個系統的使用者需要的，使用者的身份通常是家長，他們會透過輸入小孩的成績讓系統分析學習狀況與需要協助的地方，請問如果你參考附件檔案，你認為哪些資訊是家長會重視的，我希望越多越好，這樣我可以讓儀表板豐富度提升 
### 反覆確認產品功能列表
請根據我們的討論，產生一個產品功能列表，請包含所有需要的功能，並按照重要性排序

## 2. 產生系統分析文件
### 產生系統分析文件
我希望你依照家長會有多個孩子的情況，並以一個專業的系統分析師的角色幫我產生一份 DashBoard 的系統分析書，他的規格如下:
使用 docs\FormatSample\範例-系統分析.md的格式

我現在希望你重新以一個專業的系統分析師的角色，重新檢視 docs 底下的系統分析.md 文件是否符合系統分析所關注的重點與內容，若有需要請調整系統分析文件，但因為檔案會越變越大，所以請你適當的將檔案依照功能拆分成個別的 .md 並存在 docs 目錄底下，並在系統分析文件中加入檔案連結，讓後面的人可以依照連結查詢

## 3. 模組開發計劃
請依照一個專業的專案管理專家，透過讀取 docs\討論區\模組 底下所有 .md 文件後，規畫出開發階段所需的功能開發計劃與清單，放在 docs\DashBoard\phaseX 目錄底下，並且將功能開發計劃清單分為後端與前端，並關聯該功能的開發計劃檔案。
這樣做的原因是為了不要讓你日後讀取的內容太大
