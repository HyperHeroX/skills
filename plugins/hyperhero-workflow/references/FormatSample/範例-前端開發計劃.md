## 前後端協同工作流

1. 後端開發 API 並在 Swagger 中記錄
2. 執行 `pnpm api` 從後端生成 TypeScript API 客戶端
3. 前端使用生成的 API 客戶端進行開發
4. 前後端進行聯合測試

## 性能與使用者體驗考量

### 性能優化
- 懶加載組件
- 圖片優化 (壓縮、延遲加載)
- 分頁和虛擬滾動
- 代碼分割與 Tree Shaking

### 使用者體驗提升
- 表單驗證即時反饋
- 載入狀態清晰顯示
- 錯誤處理與用戶提示
- 自適應響應式設計
- 動畫過渡效果

## 測試策略

### 單元測試 (Vitest)
- 元件渲染測試
- 事件處理測試
- Store 狀態管理測試
- 表單驗證測試

### 端到端測試 (Playwright)
- 用戶流程測試
- 表單提交測試
- 頁面導航測試
- 錯誤處理測試

## 效能與品質目標

- 頁面加載時間 < 2秒
- 主 bundle 大小 ≤ 300 kB gzip
- Lighthouse 性能得分 ≥ 90
- 單元測試覆蓋率 > 80%
- 通過所有 E2E 測試
