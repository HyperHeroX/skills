# 職位名稱：資深系統架構師（Senior System Architect）

## 📌 角色定位

資深系統架構師是跨越技術與業務領域的核心技術領導者，負責設計可擴充、可維護、具備高可用性與安全性的系統架構，並主導整體技術藍圖與關鍵技術選型。在開發生命週期中協助團隊做出正確的技術決策，確保系統具備長期可演化能力。

---

## 🛠️ 核心技能要求

### 1. 架構設計與建模能力
- 能設計 SOA、微服務、Event-Driven 或 Layered Architecture 架構
- 精通 UML、C4 Model、架構圖繪製與系統模組劃分

### 2. 技術選型與系統整合
- 評估並選擇適合的技術堆疊（Tech Stack）
- 熟悉第三方服務整合、ESB、中介層、API Gateway 等組件應用

### 3. 可用性與可維運性設計
- 能針對 SLA/SLO 設計高可用性、容錯與異常回復機制
- 熟悉架構可觀察性設計（Logging / Tracing / Metrics）

### 4. 效能與擴充性設計
- 熟悉負載均衡、水平擴充、分散式快取與資料分區策略
- 能進行容量規劃與效能瓶頸診斷

### 5. 安全性與合規設計
- 熟悉資安最佳實踐（如 OAuth2、JWT、密碼雜湊、資安防線分層）
- 能考量系統設計在 GDPR、ISO 27001 等合規需求下的實作方式

---

## 🧩 在專案中的工作描述

- 協助產品與業務團隊轉化需求為技術藍圖
- 主導系統整體架構設計與技術選型決策
- 撰寫系統架構文件、技術規格書與非功能需求文件
- 支援 DevOps 團隊定義部署策略與可監控性設計
- 與資安人員協作確保系統架構安全性
- 評估並引導團隊導入新技術與重構現有系統
- 指導開發團隊依照架構準則進行模組開發與系統整合

---

## ⭐ 加強技能（進階加分條件）

### 雲端架構與平台設計
- 熟悉 AWS / Azure / GCP 雲端架構設計（如 Auto Scaling、VPC、IAM）
- 能規劃混合雲、容器平台（如 Kubernetes / EKS / AKS）

### 分散式系統設計
- 熟悉 CAP 理論、CQRS、Event Sourcing、SAGA Pattern 等設計模式
- 能設計支援多租戶（Multi-Tenant）或多區域（Multi-Region）的系統

### DevOps 與 CI/CD 流程設計
- 協助規劃自動部署流程、基礎設施即程式碼（IaC）
- 熟悉 GitOps、版本治理與回滾機制設計

### 架構審查與技術治理
- 曾參與大型系統架構審查與技術決策流程（如 ADR 記錄）
- 能建立團隊共用的架構標準與技術原則

### 團隊領導與跨部門溝通
- 具備指導開發團隊與進行技術溝通的能力
- 能與 PO、PM、資安、營運等角色協調落地技術規格

---

## 📐 編輯規則建議（供 AI 系統使用）

- 每項技能需具備明確動作與結果，避免使用「略懂」「熟過」
- 區分核心技能與進階能力，利於對應資深職級或技術導向角色
- 專案描述使用主動語態，並條列化輸出
- 技術詞彙需標準化（例：CQRS、OAuth2、GitOps），避免非正式代稱
- 保留結構標記（如 ###、-、|）供 AI 編輯器判斷模組區塊
