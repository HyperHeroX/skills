# Code Review 檢查清單
> **目標**: 突保代碼品質、可維護性和效能
> **強制要求**: 所有代碼必須通過 Code Review 才能能合併

---

## 1. TypeScript 檢查
### 1.1 型別安全
- [ ] TypeScript 嚴格模式無錯誤
- [ ] 無 `any` 型別
- [ ] 無 `@ts-ignore` / `@ts-expect-error`
- [ ] 所有變數都有明確的型別註解
- [ ] 函數參數型別完整
- [ ] 返回值型別明確
### 1.2 介面定義
- [ ] Interface 名稱清晰
- [ ] 屬性命名規範 (camelCase)
- [ ] 可選屬性標記 `?`
- [ ] 只讀屬性標記 `readonly`
- [ ] 擴展介面合理
---

### 2.1 ESLint

- [ ] 插件配置正確

## 3. 函數複雜度
### 3.1 複雜度指標
- [ ] 函數長度 < 100 行 (建議)
- [ ] 嵌套深度 < 3 層
- [ ] 參數數量 < 5 個
- [ ] 圈複雜度等級 < 10
### 3.2 單一職責
- [ ] 每個函數只做一件事
- [ ] 邊際效應明確
- [ ] 輸入/輸出清晰
---
## 4. 娡組化設計
### 4.1 關注點分離
- [ ] UI/邏輯分離
- [ ] 業務/資料分離
- [ ] 服務/倉庫庫分離
- [ ] 配置/代碼分離
### 4.2 依賴注入
- [ ] 使用 DI 容器
- [ ] 依賴注入而非硬編碼
- [ ] 介面依賴
- [ ] 易於測試
### 4.3 命名規範
- [ ] 組件: PascalCase
- [ ] 函數: camelCase
- [ ] 常量: UPPER_SNAKE_CASE
- [ ] 私有成員: _camelCase
- [ ] 布爾值: is/has + 形容詞
- [ ] 事件處理器: on + 動詞
